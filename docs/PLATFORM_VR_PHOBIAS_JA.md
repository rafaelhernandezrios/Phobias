# VR-ATR Phobias プラットフォーム — 研究機関向け概要

本ドキュメントは、プラットフォームの概要・目的・動作・連携機能を説明するものです。導入を検討する研究チーム向けに作成しています。

---

## 1. 概要

**VR-ATR Phobias** は、**VR（仮想現実）を用いた段階的恐怖曝露**を行うWebプラットフォームです。**脳波（EEG）の記録**と、**脳信号に応じた曝露レベルの自動調整**をオプションで利用できます。

- **VRET（仮想現実曝露療法）：** 被験者が恐怖対象と強度レベル（1～3）を選択し、没入型360°動画を視聴します。
- **研究利用：** 実験モードではEEGの連続記録が可能で、オプションでEEGに基づく指標（覚醒度・負荷・没入度）を用いたレベル自動調整が利用できます。
- **安全対策：** 開始時の説明と同意、常時表示の緊急終了ボタン、研究者または被験者による手動レベル調整（「強い不快感」ボタン）を備えています。

プラットフォームはブラウザ（WebXR対応VRヘッドセット含む）で動作し、**LSL（Lab Streaming Layer）**でストリームするEEGシステムや、オプションのLSLストリームによるLabRecorderなどの標準記録ツールと連携できます。

---

## 2. プラットフォームの機能

### 2.1 利用者（被験者）の流れ

1. **開始画面：** 法的注意事項と同意の表示。続行には同意が必要です。
2. **メニュー：** 選択可能な恐怖症（クモ・閉所・高所・ヘビ・昆虫など）を表示。被験者が1つ選択します。
3. **曝露レベル：** 各恐怖症に3段階（軽度・中等度・強度）。被験者が手動で選択するか、**実験モード**ではレベルが自動または研究者によって調整されます。
4. **VR曝露：** 360°動画を再生。一時停止・再開・終了はいつでも可能。**EMERGENCY EXIT（緊急終了）**ボタンで即座にセッションを終了できます。
5. **終了：** メニューへ戻るか、アプリを閉じます。

### 2.2 二つの利用モード

| モード | 説明 | 曝露レベル |
|--------|------|------------|
| **標準** | メニューから恐怖症とレベル（1～3）を選択。360°再生のみ。 | 固定（被験者が選択） |
| **EEG実験** | 脳波記録を伴う研究向け。レベルは**EEGに基づく自動調整**または研究者の手動操作が可能。 | 可変：EEG自動／PC手動／併用 |

実験モードではセッションイベントが記録され、EEGを利用する場合はタイムスタンプとレベルラベル付きでCSVに保存されます。

## 3. 全体フロー（EEG実験モード）

EEGと適応レベルを用いる場合の典型的なデータ・制御の流れです。

```
┌─────────────────────────────────────────────────────────────────────────┐
│  被験者（ブラウザまたはVRヘッドセット）                                   │
│  • 同意 → 恐怖症選択 → 実験モード開始                                     │
│  • 360°動画視聴；レベル(1～3)は自動または研究者により変更可能              │
│  • 「強い不快感」（レベル低下）または「EMERGENCY EXIT」（終了）を押下可能   │
└─────────────────────────────────────────────────────────────────────────┘
                    │                              ▲
                    │ WebSocket（イベント・レベル） │ WebSocket（adaptive_state, force_level）
                    ▼                              │
┌─────────────────────────────────────────────────────────────────────────┐
│  Webサーバ（HTML/JS、必要に応じてNode/Python）                            │
│  • アプリを配信（VR/WebXR用にHTTPS）                                      │
│  • experiment.html が適応指標を受信しレベルルールを適用                   │
└─────────────────────────────────────────────────────────────────────────┘
                    │
                    │ WebSocket（同一サーバまたは同一PC）
                    ▼
┌─────────────────────────────────────────────────────────────────────────┐
│  レコーダ（Python: aura_recorder.py）                                      │
│  • EEG装置からLSL受信（例：AURA、8ch、約250Hz）                           │
│  • EEG＋ラベルをCSVで保存（output/）                                     │
│  • 2秒ごとにFear/Engagement指標を計算し "adaptive_state" を送信           │
│  • "manual_level"（GUI等から）を受信し "force_level" を転送              │
│  • オプション --lsl：状態をLSLに公開、手動レベルをLSLで受信               │
└─────────────────────────────────────────────────────────────────────────┘
        │                    │
        │ LSL（EEG）          │ LSL（オプション）: VRPhobia_State, VRPhobia_ManualLevel
        ▼                    ▼
┌──────────────┐    ┌─────────────────────────────────────────────────────┐
│  EEG装置     │    │  PCモニタ（adaptive_monitor_gui.py）                  │
│  （例：AURA）│    │  • 指標・メトリクスをリアルタイム表示                 │
│              │    │  • Level 1 / 2 / 3 ボタンでシーン変更                 │
└──────────────┘    └─────────────────────────────────────────────────────┘
```

**フロー概要：**

1. **被験者**がブラウザまたはVRでアプリを開き（HTTPS）、同意し、恐怖症を選び、実験モードを開始します。
2. **レコーダ**（Python）はLSLでEEG装置に接続し、WebSocketでブラウザと接続。EEGをレベルラベル付きでCSVに保存し、適応モード時は2秒ごとにEEGから指標を計算してレベル提案（上げる・維持・下げる）を送信します。
3. **Webアプリ**はその提案を受け、安全ルール（ヒステリシス・クールダウン）を満たす場合に動画レベルを変更します。被験者は「強い不快感」でいつでもレベルを下げられます。
4. **研究者**は**PCモニタ**で脳状態をリアルタイムに確認し、必要に応じてLevel 1／2／3で手動変更できます。変更は直ちに被験者のシーンに反映されます。
5. オプションで、レコーダは状態（指標・現在レベル）をLSLに公開し、LSLでレベルコマンドを受信し、BCIやLabRecorderなど他システムと連携できます。

## 4. コンテンツ：恐怖症とレベル

**5種類の恐怖症**を用意し、それぞれ**3段階の強度**（別々の360°動画）に対応しています。内容はデータファイル（`app/data/content.json`）で定義され、動画・時間・テキストは研究機関で変更可能です。

| 恐怖症 | 概要 | レベル例 |
|--------|------|----------|
| クモ恐怖症（Arachnophobia） | クモへの恐怖 | 1: 軽度 / 2: 中等度 / 3: 強度（接近・複数） |
| 閉所恐怖症（Claustrophobia） | 閉鎖空間への恐怖 | 1: 小部屋 / 2: エレベーター / 3: トンネル・壁が迫る |
| 高所恐怖症（Acrophobia） | 高所への恐怖 | 1: 低いバルコニー / 2: 高層建築 / 3: 縁・橋 |
| 蛇恐怖症（Ophidiophobia） | ヘビへの恐怖 | 1: 遠距離 / 2: 近くで動く / 3: 接近してくる |
| 昆虫恐怖症（Entomophobia） | 昆虫への恐怖 | 1: 1匹の小さい昆虫 / 2: 複数 / 3: 群れ・多数 |

**ベースライン**（中性シーン）も定義でき、較正やEEGベースラインに利用できます。動画は差し替え可能で、`content.json` のURLとメタデータの設定のみ必要です。

## 5. 技術的連携

### 5.1 脳波（EEG）

- **プロトコル：** LSL（Lab Streaming Layer）。リアルタイム信号研究で標準的に利用されます。
- **想定装置：** AURA（8ch、約250Hz）。チャンネルマッピングを調整すれば、他のLSL対応装置も利用可能です。
- **推奨モンタージュ（8電極、10–20法）：** F3, F4, Fz, Cz, Pz, P3, P4, Oz。前頭 theta・beta/alpha・後部 alpha 抑制・前頭非対称性を用いた複合指標でレベル適応が可能です。
- **出力：** `output/` フォルダにCSV（タイムスタンプ・各ch値・ラベル（恐怖症＋レベル））。他ツールとの同期のためLSLタイムスタンプをそのまま使用します。

### 5.2 EEG適応レベル

- **Fear/Engagement指標：** EEG帯域（theta, alpha, beta）と前頭非対称性の組み合わせ。セッション開始時に推定したベースラインで正規化します。
- **ルール：** 指標が中程度の範囲ならレベル上昇（1→2）を提案、閾値を超えた場合または「強い不快感」押下時はレベル低下（2/3→1/2）。振動を防ぐためヒステリシスと最小間隔を設けています。
- **技術詳細：** [EEG_ADAPTIVE_LEVELS.md](EEG_ADAPTIVE_LEVELS.md)（式・モンタージュ・閾値）。

### 5.3 PCモニタ（研究者用）

- **アプリ：** PythonのGUIスクリプト（`scripts/adaptive_monitor_gui.py`）。
- **機能：** Fear/Engagement指標・レベル提案・現在レベル・EEGメトリクスをリアルタイム表示。Level 1／2／3ボタンで手動変更。変更はWebSocketでレコーダに送られ、ブラウザに転送されてVRシーンが更新されます。
- **利用：** レコーダと同じマシン（またはネットワーク上）で実行。実験がHTTPSの場合は `--wss` を指定します。

### 5.4 LSL拡張（オプション）

レコーダの `--lsl` オプション使用時：

- **VRPhobia_State（出力ストリーム）：** レコーダが現在状態（恐怖/没入指標・現在レベル）をLSLに公開。LabRecorderなどLSL対応アプリでEEGと一緒に記録できます。
- **VRPhobia_ManualLevel（入力ストリーム）：** 値 1, 2, 3 を受け付けるLSLストリームをレコーダが監視。他アプリ（スクリプト・BCI等）がLSLでレベルを送ると、モニタのボタンと同様にVRシーンが更新されます。

既存のLSL研究パイプラインに組み込むことができ、ブラウザや同梱モニタに依存しない連携が可能です。

### 5.5 イベントログ

- 同意・恐怖症選択・レベル・動画開始/終了・一時停止・緊急終了などのユーザー操作が、セッションID・タイムスタンプ付きでブラウザ内に記録されます。JSON形式でコンソールからエクスポートし、後から解析できます。

## 6. 安全と倫理

- **同意：** メニューに進む前に説明を表示し、明示的な同意を求めます。
- **緊急終了：** 曝露中は **EMERGENCY EXIT** ボタンを常時表示。押下でセッション終了しメニューへ戻ります。
- **強い不快感：** 実験モードでは被験者がいつでもレベルを下げられ、セッションを抜けずに済みます。
- **研究者による制御：** 研究者はPCモニタ（またはLSL）からレベルを上げ下げできます。

本プラットフォームは、研究の倫理設計（プロトコル・被験者説明・委員会承認）に代わるものではありません。各機関の規程に合わせて手順と同意書を定めてください。

## 7. 技術要件（概要）

| 構成要素 | 要件 |
|----------|------|
| **ブラウザ／VR** | WebXR対応のモダンブラウザ（例：Chrome）。VRヘッドセットはHTTPSでアクセス（サーバと同一ネットワーク）。 |
| **サーバ** | アプリを配信する静的Webサーバ（Node、Python等）。VRを用いる実験モードではHTTPS推奨（自己署名または正式証明書）。 |
| **EEG** | LSL対応装置（例：AURA）。レコーダと同じマシンまたはネットワークでストリームしていること。 |
| **Python（レコーダ）** | Python 3.8以上。依存: `pylsl`, `websockets`, `numpy`, `scipy`（適応レベル用）。 |
| **モニタ（任意）** | Python と `websockets`、GUI環境（tkinter）。 |

EEG実験のインストール・実行手順は [EEG_EXPERIMENT_SETUP.md](EEG_EXPERIMENT_SETUP.md) を参照してください。

## 8. 研究用データ出力

- **EEG CSV：** セッションごとに `output/` に1ファイル。列：LSLタイムスタンプ、ch1～ch8（または装置に応じたch）、ラベル（例：`arachnophobia_level2`）。LSLおよび他記録と同期可能です。
- **VRPhobia_State LSLストリーム（オプション）：** レコーダの更新レート（約0.5Hz）で指標と現在レベル。LabRecorderでの記録やリアルタイム解析に利用できます。
- **セッションログ：** アプリイベントをJSONでエクスポート（同意・恐怖症・レベル・タイムスタンプ・操作）。行動解析やプロトコル遵守の確認に有用です。
